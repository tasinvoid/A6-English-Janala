Frequently Asked Questions

Whats the difference between var, let, and const?
Var is function-scoped, hoisted, and allows redeclaration. Let is block-scoped and can be reassigned but not redeclared. Const is block-scoped and can't be reassigned, but object contents can change. Use const by default, let when needed, and avoid var.
Whats the difference between map(), forEach(), and filter()?
Map returns a new array by transforming each element. forEach runs a function on each element but returns nothing. Filter creates a new array with elements that pass a test. Map and filter are pure, forEach is for side effects.
Explain arrow functions and how they are different from regular functions
Arrow functions are shorter syntax with implicit returns. They don't have their own 'this', 'arguments', or 'super', inheriting from parent scope instead. Can't be constructors or methods. Great for callbacks and one-liners but less flexible than regular functions.
How JavaScript Promises work?
Promises handle async operations. They're objects with pending, fulfilled, or rejected states. Create with new Promise(). Chain with .then() for success and .catch() for errors. Can compose multiple promises with Promise.all() or Promise.race(). Async/await makes them look synchronous.
How closures work in JavaScript?
Closures happen when functions remember the environment where they were created. Inner functions can access outer function variables even after the outer function finishes. They're used for data privacy, factory functions, and maintaining state. Basically, they keep variables alive when they'd normally disappear.
